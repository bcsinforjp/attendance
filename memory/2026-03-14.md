**Problem:** Panel 20 ("🔍 Recent Records" / "Attendance Detail") had a SQL syntax error:
- Original query used `BETWEEN $__timeFrom AND $__timeTo` which Grafana's PostgreSQL datasource couldn't interpolate properly
- Error: `syntax error at or near "$"`

**Fix:** Changed to `$__timeFilter(record_date)` macro instead. This is the correct Grafana macro for PostgreSQL time filtering.

### Fix History (3 iterations)

**Fix 1 — Time filter:** `BETWEEN $__timeFrom AND $__timeTo` → `$__timeFilter(record_date)`

**Fix 2 — Employee filter (single):** `'${employee}'` → `${employee:sqlstring}` with `IN` clause (proper quoting for varchar)

**Fix 3 — All + Multi-select:** Added custom "All" option to variable query + used `:csv` format with `STRING_TO_ARRAY`

### Final working query (Panel 20):
```sql
SELECT personal_code AS "ID",
       full_name AS "Name",
       record_date AS "Date",
       commute_time AS "Clock In",
       time_to_leave AS "Clock Out",
       working_hours AS "Work Hours"
FROM attendance_records
WHERE $__timeFilter(record_date)
  AND ('${employee:csv}' = 'All' OR personal_code = ANY(STRING_TO_ARRAY('${employee:csv}', ',')))
ORDER BY record_date DESC
LIMIT 200
```

### Updated variable query:
```sql
SELECT 'All' AS __value, '-- All Employees --' AS __text
UNION ALL
SELECT DISTINCT personal_code AS __value, personal_code || ' - ' || full_name AS __text
FROM attendance_records ORDER BY 2
```
