# 2026-03-13 - Grafana ROUND() Fix

## Problem
PostgreSQL 17 doesn't support `ROUND(double precision, integer)` — requires explicit `::numeric` cast.

## Solution
Fixed 7 panels across both dashboards:
- **Attendance Management** (3 panels): Avg Hours, Avg Hours by Day, Top 10 Workers
- **Employee Analytics** (4 panels): Avg Hours/Employee, Top 10 by Hours, Daily Work Trend, Monthly Summary

Changed: `ROUND(AVG(expr), 2)` → `ROUND((AVG(expr))::numeric, 2)`

## Status
✅ All queries working. Both dashboards live at http://192.168.0.2/grafana/

User asked about switching to InfluxDB — advised against it for relational data. Agreed to fix SQL issues first.
